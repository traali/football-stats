## Critical: DESIGN.md Internal Contradiction on Radius Tokens

DESIGN.md §12 defines radius tokens, but §6.2 Card spec contradicts them:

| Token | §12 Token Spec | §6.2 Card Spec | CSS Implementation |
|-------|---------------|----------------|-------------------|
| `--radius-xl` | 16px | `rounded-xl (12px)` | **16px** |
| `--radius-2xl` | 20px | `rounded-2xl (16px)` | **20px** |

CSS follows §12, meaning cards render with `rounded-xl = 16px` instead of the documented 12px. **Fix DESIGN.md §6.2 to match §12, or vice versa.**

---

## High: Missing Spacing Tokens in CSS

DESIGN.md §5.1 defines `--space-1` through `--space-9` (4px–96px grid). These are **completely absent** from `index.css`. All spacing is done via hardcoded Tailwind utilities (`p-3`, `p-5`, `gap-0.5`, etc.) — 78 occurrences across components. No CSS custom properties back these.

**Impact:** Spacing is not tokenized. Any global spacing change requires editing every component individually.

---

## Medium: Hardcoded `text-[10px]` (8 occurrences)

The `text-[10px]` arbitrary value is used in 8 places across components. DESIGN.md §4.2 defines `--text-caption` (12px) and `--text-micro` (10px) roles but no CSS custom properties exist for them. These should either:
- Be a named token (e.g., `--font-size-micro: 10px`)
- Or at minimum be a Tailwind utility if spacing tokens are adopted

Files: `MatchHeader.tsx`, `BottomNav.tsx`, `Home.tsx`, `StatBadge.tsx`, `StandingsTable.tsx`, `PlayerCard.tsx`, `MatchPage.tsx`

---

## Medium: `drop-shadow` Violates No-Shadow Principle

`BottomNav.tsx:31` applies `drop-shadow-[0_0_6px_var(--color-accent-glow)]` on active icons. DESIGN.md §7 explicitly states "No shadows ever" and §12 says `/* Shadows (NONE — surface ladder replaces shadows) */`. This is the only shadow in the codebase but it contradicts the design spec.

**Recommendation:** Replace with `text-shadow` (CSS only, no box-shadow) or rely on accent color alone for active state.
