# Code Review: TS Strictness

## HIGH: Unused import `APP_CONFIG` in dataProcessors.ts
- **File**: `src/utils/dataProcessors.ts` :1
- **Issue**: `APP_CONFIG` is imported from `'../types/config'` but never referenced. The function `processPlayerMatchHistory` receives `currentSeasonId` and `previousSeasonId` as explicit parameters rather than using the config constant.
- **Suggestion**: Remove the `APP_CONFIG` import on line 1 — it is dead code.

## HIGH: `any` type parameter defeats strict checking
- **File**: `src/pages/TeamPage.tsx` :65
- **Issue**: `getCategoryName` declares `(c: any): string | null`. The `any` type bypasses all TypeScript strictness checks and spreads unchecked access throughout the function body (`c.category_name`, `c.category_name_translations.fi`, `c.competition_id`, etc.).
- **Suggestion**: Replace `any` with `Record<string, unknown>` and use proper type narrowing, or define a union of the known shapes from the `TeamResponse` categories/primary_category fields.

## HIGH: Unsafe type assertion in PlayerPage
- **File**: `src/pages/PlayerPage.tsx` :63
- **Issue**: `(player as Record<string, unknown>).player_statistics as Array<Record<string, string>> | undefined` uses a double `as` assertion to bypass type safety. The `PlayerAPIResponse` type has an index signature `[key: string]: unknown` but does not declare `player_statistics`. This means any structural change to the API response would silently pass through.
- **Suggestion**: Either add `player_statistics?: Record<string, string>[]` to the `PlayerAPIResponse` interface in `src/types/api.ts`, or use a type-safe accessor function that validates the shape at runtime.

## MEDIUM: Duplicate destructuring creates unused variables
- **File**: `src/components/CommonOpponents.tsx` :24
- **Issue**: Props destructuring includes both `teamBId` (shorthand) and `teamBId: _` (alias), and both `teamAName` (shorthand) and `teamAName: __` (alias). The `_` and `__` variables are never used. This is confusing and, with `noUnusedLocals` enabled, would produce compile errors.
- **Suggestion**: Remove the duplicate `teamBId: _` and `teamAName: __` aliases. The original shorthand bindings already provide the values used throughout the component.
